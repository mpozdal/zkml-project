Master's thesis project

About this project

ZKML Credit Classifier demonstrates Zero-Knowledge Machine Learning (ZKML) in consumer credit scoring. A logistic regression model evaluates creditworthiness while sensitive financial inputs stay on the client. Only a Groth16 proof and the public decision bit are sent to Ethereum (Sepolia).

What it does

  • You enter four features from the German Credit dataset schema (duration, amount, installment commitment, age).
  • The browser applies the same StandardScaler and fixed-point encoding (SCALE = 1000) as the Python training pipeline.
  • SnarkJS generates a ZK-SNARK proof locally (WebAssembly) that the Circom circuit computed the model output correctly.
  • MetaMask submits the proof to CreditRegistry, which verifies it on-chain and stores only approved / denied plus a timestamp.

System architecture

The repository is split into four modules:

ml-model/ — Python, scikit-learn
Trains logistic regression on German Credit data; exports quantized weights, scaler parameters, and model_config.json.
zero-knowledge/ — Circom, SnarkJS
Circuit replicates fixed-point inference (dot product with // SCALE); builds credit_classifier.wasm and Groth16 proving keys.
blockchain/ — Solidity, Hardhat Ignition
Groth16Verifier.sol checks proofs; CreditRegistry.sol anchors decisions per wallet and application ID.
web/ — Angular, Tailwind, ethers.js
SPA for wallet connection, preprocessing, in-browser proof generation, and Sepolia transactions.

Data flow

Form (raw features)
  → StandardScaler + SCALE=1000 (model_config.json)
  → Unsigned 32-bit encoding (two's complement for Circom Num2Bits)
  → Circom private inputs x[4]
  → Groth16 proof + public signal y (0 or 1)
  → CreditRegistry.anchorCreditDecision()
  → On-chain: isApproved, timestamp, applicationId

Raw form values are never sent as plain calldata. The chain sees the proof, the public decision, and registry metadata—not your inputs or model weights.

Privacy & security

Verify, don't trust

The verifier checks the Groth16 proof against the deployed circuit—no server-side scoring.

Private witness

x[4] are private Circom inputs; only public output y is revealed in the proof.

Pipeline parity

Angular preprocessing matches ml-model/train.py (same scaler, weights, fixed-point division). Unit tests assert the documented example row produces decision 1.

On-chain storage

Per (userAddress, applicationId) the registry stores:

  • isApproved — model decision (0 or 1)
  • timestamp — block time
  • exists — replay protection

Proofs, raw features, and weights are not persisted on-chain.
